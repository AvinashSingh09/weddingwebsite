# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Active Prototype Direction

- Selected concept: Royal Chronicle.
- Preserve: full-bleed Jaisalmer dusk ceremony, near-black editorial shell, antique-gold rules and iconography, high-contrast serif display type, open event rail, split story/passport section, fixed guest utility dock.
- Avoid: purple, generic dashboard cards, bright gradients, glassmorphism, rounded SaaS components, decorative clutter.
- Product goal: a responsive, interactive destination wedding guest companion with logistics, RSVP, passport, memories, notes, polls, and post-wedding legacy.
- Opening experience: on initial page load, play the supplied intro film full-screen before revealing the main site; preserve a reliable skip path and mobile-safe playback.
